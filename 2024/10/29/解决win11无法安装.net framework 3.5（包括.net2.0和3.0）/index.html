解决 Win11 无法安装 .NET Framework 3.5（包括 .NET 2.0 和 3.0）

发表于 2024-10-29 更新于 2026-04-13 分类于 数据库

安装 SQL Server 时，系统提示需要安装 .NET Framework 3.5，但按照提示安装后却一直失败。本文记录了尝试过的所有方法，以及最终解决方案。

问题背景

按照网上教程下载 SQL Server 安装包后无法打开，系统提示需要安装 .NET Framework 3.5（包括 .NET 2.0 和 3.0），但安装始终失败。

可以通过以下路径查看是否安装成功：

控制面板 → 程序 → 启用或关闭 Windows 功能

尝试过的方法

方法一：通过 Windows 功能启用

  1. 打开控制面板，搜索并选择「Windows 功能」
  2. 找到并勾选「.NET Framework 3.5 (包括 .NET 2.0 和 3.0)」
  3. 点击确定，等待系统下载安装

方法二：使用 DISM 命令行工具（本地安装包）

如果已有 .NET Framework 3.5 的安装包，可以通过 DISM 离线安装：

  1. 将安装包解压到某个目录，例如 D:\sxs

方法四：下载离线安装包

从网上下载 .NET Framework 3.5 离线安装包，配合方法二的 DISM 命令进行安装。
